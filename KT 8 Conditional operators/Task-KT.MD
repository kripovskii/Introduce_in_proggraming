### Задание: Система оценки риска кредита 

Программа должна:

1. Запросить следующие данные (все через `Console.ReadLine()`):

   - Возраст (int)  
   - Ежемесячный официальный доход (double, руб.)  
   - Желаемая сумма кредита (double, руб.)  
   - Срок кредита в месяцах (int)  
   - Есть ли текущая просрочка? ("да" / "нет")  
   - Сколько действующих кредитов (int ≥ 0)  
   - Семейное положение (строка: "холост", "женат", "замужем", "разведен", "вдовец")  
   - Есть ли дети до 18 лет? ("да" / "нет")

2. **Проверить корректность всех введённых данных**.  
   При **любой** ошибке — вывести понятное сообщение.

   Правила валидации:
   - Возраст: 18–80 включительно  
   - Доход: ≥ 0  
   - Сумма кредита: > 0  
   - Срок: от 3 до 60 месяцев включительно  
   - Просрочка: только "да" или "нет" (игнорировать регистр)  
   - Кол-во кредитов: ≥ 0  
   - Семейное положение: только один из вариантов выше (игнорировать регистр)  
   - Дети: только "да" или "нет" (игнорировать регистр)

3. Посчитать **скоринговый балл** (int) по следующим правилам (накапливаем в переменную `score`):

| Условие                                    | Δ балла   |
|--------------------------------------------|-----------|
| Возраст < 21                               | -25       |
| 21 ≤ Возраст ≤ 25                          | -10       |
| 26 ≤ Возраст ≤ 45                          | +15       |
| 46 ≤ Возраст ≤ 60                          | +5        |
| Возраст > 60                               | -15       |
| Доход < 30000                              | -40       |
| 30000 ≤ Доход < 60000                      | -10       |
| 60000 ≤ Доход < 100000                     | +10       |
| Доход ≥ 100000                             | +30       |
| Есть просрочка                             | -80       |
| Действующих кредитов ≥ 3                   | -35       |
| Есть дети                                  | -10       |
| Семейное положение = "женат" или "замужем" | +20       |
| Сумма кредита > доход × 12                 | -50       |
| Сумма кредита ≤ доход × 3                  | +25       |
| Срок > 36 месяцев                          | -15       |

4. На основе итогового балла определить решение и примерную ставку:

| Балл              | Решение                  | Ставка         | Ключевое сообщение                                      |
|-------------------|--------------------------|----------------|-----------------------------------------------------------------|
| ≤ -100            | Отказ                    | —              | Кредит не может быть одобрен                                    |
| от -99 до -41     | Отказ с рекомендацией    | —              | Кредит отклонён. Рекомендуем закрыть просрочки / повысить доход |
| от -40 до 0       | Одобрено с условиями     | 29–35%         | Одобрено под высокую ставку                                     |
| от 1 до 40        | Одобрено                 | 19–28%         | Кредит одобрен                                                  |
| от 41 до 80       | Одобрено + бонус         | 11–18%         | Хорошая кредитная история — приятная ставка                     |
| > 80              | VIP-одобрение            | 7–10%          | Поздравляем! Вы наш лучший клиент                               |

5. Вывести **дополнительные рекомендации** (независимо от решения):

   - Если просрочка == "да" → «Погасите все просроченные платежи как можно скорее»  
   - Если доход < 40000 && сумма > 300000 → «Рассмотрите уменьшение запрашиваемой суммы»  
   - Если возраст > 65 && срок > 24 → «Желательно сократить срок кредита»  
   - Если балл > 60 && просрочки нет → «Доступна программа лояльности: -2% ставки при оформлении страховки»  

6. **Требования именно к условным операторам** (обязательно использовать):

   - Многоступенчатые конструкции `if – else if – else` (минимум для возраста и для дохода)  
   - Конструкцию `switch` — хотя бы для определения семейного положения **и** для выбора итогового решения/ставки по диапазону балла  
   - Логические операторы `&&`, `||`, `!` — активно и в сложных условиях  
   - Вложенные `if` — хотя бы в 2–3 местах
