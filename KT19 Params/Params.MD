
## Ключевое слово `params` в C#

### 1. Введение

В C# часто возникает необходимость передавать **переменное количество аргументов** в метод. Например, вы хотите написать метод, который может складывать любое количество чисел — два, три, десять и даже ноль.

Для этого в C# существует специальное ключевое слово: **`params`**.

---

### 2. Что такое `params`?

Ключевое слово **`params`** позволяет методу принимать **переменное число аргументов** одного типа. Под капотом эти аргументы передаются как **массив** (`T[]`).

####  Основные правила:
- `params` может быть только **последним параметром** в списке параметров метода.
- Допускается **только один** параметр с `params`.
- Все аргументы, передаваемые через `params`, должны быть одного типа.

---

### 3. Простой пример

```csharp
public static int Sum(params int[] numbers)
{
    int total = 0;
    foreach (int number in numbers)
    {
        total += number;
    }
    return total;
}
```

Использование:

```csharp
Console.WriteLine(Sum(1, 2, 3));         // 6
Console.WriteLine(Sum(10, 20, 30, 40));  // 100
Console.WriteLine(Sum());                // 0 (пустой массив)
```

Здесь компилятор автоматически создаёт массив `int[]` из переданных значений.

---

### 4. Использование с другими параметрами

`params` может сочетаться с обычными параметрами, **но только если он идёт последним**:

```csharp
public static void LogMessage(string prefix, params string[] messages)
{
    Console.WriteLine($"[{prefix}]");
    foreach (var msg in messages)
    {
        Console.WriteLine($"  - {msg}");
    }
}
```

Пример вызова:

```csharp
LogMessage("INFO", "Пользователь вошёл", "Сессия начата");
```

Вывод:
```
[INFO]
  - Пользователь вошёл
  - Сессия начата
```

---

### 5. Передача массива напрямую

Вы также можете передать уже существующий массив:

```csharp
int[] values = { 1, 2, 3, 4, 5 };
int result = Sum(values); // работает!
```

⚠️ Важно: если вы передаёте массив, он **не оборачивается в другой массив** — используется напрямую.

---

### 6. `params` с типом `object` или `string`

Можно использовать `params object[]`, но будьте осторожны с **боксингом** (boxing) при передаче значимых типов.

Пример:

```csharp
public static void PrintAll(params object[] items)
{
    foreach (var item in items)
        Console.WriteLine(item);
}

PrintAll("Привет", 42, DateTime.Now);
```

---

### 7. Практическое применение

- Методы форматирования строк (`string.Format`, `Console.WriteLine`).
- Логгеры.
- Математические утилиты (сумма, среднее, мин/макс).
- Конструкторы или фабрики, принимающие переменное число элементов.

---

### 8. Ограничения и советы

| Ограничение | Примечание |
|------------|-----------|
| Только один `params` | Нельзя иметь два `params` параметра |
| Последний в списке | Иначе ошибка компиляции |
| Производительность | Использование `params` приводит к выделению массива в куче — может быть накладно в горячих путях |
| Альтернативы | Для высокопроизводительного кода используйте `ReadOnlySpan<T>` или перегрузки методов |

>  **Совет**: В .NET 6+ и C# 11 можно использовать `params Span<T>` или `params ReadOnlySpan<T>` для избежания аллокаций на стеке / куче (но это продвинутая тема).

---

### 9. Заключение

Ключевое слово `params` — мощный и удобный инструмент для создания **гибких методов** с переменным числом аргументов. Оно делает API более удобным для вызова и улучшает читаемость кода.

Но помните: удобство ≠ производительность. В критичных участках кода лучше использовать альтернативы.

---

###  Кратко:
- `params T[]` → передаём сколько угодно `T`.
- Только один и только последний параметр.
- Компилятор сам создаёт массив.
- Отлично подходит для утилит, логгеров, форматтеров.
