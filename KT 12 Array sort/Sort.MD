# Простая сортировка (Simple Sort)

Термин **«простая сортировка»** в контексте алгоритмов чаще всего используется как **разговорное или обобщённое название** для самых элементарных методов упорядочивания массивов — таких как:

- **Сортировка выбором (Selection Sort)**
- **Сортировка вставками (Insertion Sort)**
- **Пузырьковая сортировка (Bubble Sort)**

Однако в некоторых учебных источниках (особенно на русском языке) под **«простой сортировкой»** могут подразумевать именно **сортировку выбором**, так как её логика интуитивно понятна:
> *«Найди самый маленький элемент и поставь его первым, затем следующий — вторым, и так далее»*.

---

##  Алгоритм: Сортировка выбором (Selection Sort)

###  Идея
На каждом шаге алгоритм:
1. Находит **минимальный элемент** в неотсортированной части массива.
2. Меняет его местами с **первым элементом** неотсортированной части.
3. Сдвигает границу отсортированной части вправо на один элемент.

Повторяется до тех пор, пока весь массив не будет отсортирован.

---

##  Пошаговый пример

Исходный массив:
`[64, 25, 12, 22, 11]`

| Шаг | Неотсортированная часть | Минимум | Обмен                     | Результат после шага     |
|-----|--------------------------|---------|---------------------------|--------------------------|
| 1   | [64, 25, 12, 22, 11]     | 11      | 64 ↔ 11                   | **[11]**, 25, 12, 22, 64 |
| 2   | [25, 12, 22, 64]         | 12      | 25 ↔ 12                   | **[11, 12]**, 25, 22, 64 |
| 3   | [25, 22, 64]             | 22      | 25 ↔ 22                   | **[11, 12, 22]**, 25, 64 |
| 4   | [25, 64]                 | 25      | уже на месте              | **[11, 12, 22, 25]**, 64 |
| 5   | [64]                     | 64      | —                         | **[11, 12, 22, 25, 64]** |

Готово!

---

##  Реализация на C#

```csharp
public static void SimpleSort(int[] arr)
{
    if (arr == null || arr.Length <= 1)
        return;

    int n = arr.Length;

    for (int i = 0; i < n - 1; i++)
    {
        // Предполагаем, что минимальный элемент — первый в неотсортированной части
        int minIndex = i;

        // Ищем реальный минимум
        for (int j = i + 1; j < n; j++)
        {
            if (arr[j] < arr[minIndex])
                minIndex = j;
        }

        // Меняем местами, только если найден меньший элемент
        if (minIndex != i)
        {
            (arr[i], arr[minIndex]) = (arr[minIndex], arr[i]); // C# 7+ tuple swap
        }
    }
}
```

### Пример использования:

```csharp
int[] data = { 64, 25, 12, 22, 11 };
SimpleSort(data);
// data = [11, 12, 22, 25, 64]
```

## Характеристики алгоритма

| Параметр               | Значение                     |
|------------------------|------------------------------|
| **Временная сложность**| **O(n²)** во всех случаях    |
| **Лучший случай**      | O(n²) (всё равно ищем минимум на каждом шаге) |
| **Худший случай**      | O(n²)                        |
| **Средний случай**     | O(n²)                        |
| **Дополнительная память** | **O(1)** (in-place)       |
| **Стабильность**       | ❌ Нет (может нарушать порядок равных элементов) |
| **Адаптивность**       | ❌ Нет (не ускоряется на частично отсортированных данных) |

---

##  Преимущества

- Простота реализации и понимания.
- Минимальное количество обменов: **ровно (n − 1)** в худшем случае — это меньше, чем у Bubble или Insertion Sort.
- Подходит для ситуаций, где **запись в память дорога** (например, в embedded-системах), но чтение — дёшево.

---

##  Недостатки

- Квадратичная сложность делает его **непригодным для больших массивов**.
- Не стабилен — может менять относительный порядок одинаковых элементов.
- Не адаптивен — не использует преимущества уже отсортированных участков.

---

##  Сравнение с другими "простыми" сортировками

| Алгоритм           | Обмены      | Стабилен? | Адаптивен? | Практическое применение |
|--------------------|-------------|-----------|------------|--------------------------|
| **Selection Sort** | O(n)        | ❌        | ❌         | Очень редко              |
| **Bubble Sort**    | O(n²)       | ✅        | ✅ (с оптимизацией) | Почти никогда       |
| **Insertion Sort** | O(n²)       | ✅        | ✅         | Для малых/почти отсортированных массивов |

> **Insertion Sort** на практике почти всегда **быстрее**, чем Selection Sort, несмотря на ту же асимптотику.

---

##  Заключение

**«Простая сортировка»**, как правило, — это **сортировка выбором**.
Она идеально подходит для:
- Обучения основам алгоритмов,
- Понимания концепции «разделяй и властвуй» на примитивном уровне,
- Сценариев с **очень маленькими массивами** (n < 10).

Однако в реальных приложениях предпочтение стоит отдавать:
- `Array.Sort()` — для общего случая,
- `Insertion Sort` — для малых или почти отсортированных данных.

> 💡 **Запомните**: «Простая» ≠ «Хорошая».

## P.s сортировка которую показывал на паре:

```csharp
using System;
using System.Collections.Concurrent;

namespace ConsoleApplication1
{
    internal class Program
    {
        static void Main(string[] args)
        {
            int[] array = new int[10];
            for (int i = 0; i < array.Length; i++)
            {
                Random random = new Random();
                array[i] = random.Next(100);
                Console.WriteLine("array[{0}] = {1}", i, array[i]);
            }

            int j = 0;
            while (j < array.Length - 1)
            {
                if (array[j] > array[j + 1]) //[j;j+1]
                {
                    int temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                    j = 0;
                }
                else j++;
            }
            for (int i = 0; i < array.Length; i++)
            {
                Console.WriteLine("array[{0}] = {1}", i, array[i]);
            }
        }
        
    }
}
```
