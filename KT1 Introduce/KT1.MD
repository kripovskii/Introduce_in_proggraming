# История программирования и язык C#

## Эволюция программирования: Краткий обзор

История программирования — это путь от машинных кодов и перфокарт до высокоуровневых языков и облачных вычислений.

### Ключевые вехи:

1.  **1843:** Ада Лавлейс публикует первый в мире алгоритм для аналитической машины Чарльза Бэббиджа, что делает её первым программистом.
2.  **1940-е:** Создаются первые электронные компьютеры (ENIAC). Программирование велось на машинном коде (бинарные команды) или с помощью проводных панелей.
3.  **1950-е:** Появление первых языков высокого уровня: **Fortran** (для научных вычислений), **LISP** (для искусственного интеллекта) и **COBOL** (для бизнес-задач). Это позволило абстрагироваться от аппаратной части.
4.  **1970-е:** Эра структурного программирования. Языки **C** (Деннис Ритчи) и **Pascal** (Никлаус Вирт) стали доминировать благодаря своей эффективности и четкой структуре.
5.  **1980-е:** Рождение объектно-ориентированного программирования (ООП). **C++** (Бьёрн Страуструп), как развитие языка C, и **Objective-C** добавили мощные возможности ООП.
6.  **1990-е:** **Эра Интернета**. Язык **Java** (James Gosling) с его философией "Write Once, Run Anywhere" (WORA) стал невероятно популярным. Появляются скриптовые языки для веба: **JavaScript**, **PHP**, **Python**.
7.  **2000-е по н.в.:** Доминирование управляемого кода, веб-фреймворков и скриптовых языков. Рост популярности функционального программирования. Появление языков для специфичных задач: **Go** (от Google), **Rust** (от Mozilla), **Swift** (от Apple).

## Язык C#

C# (произносится как "Си Шарп") — это современный, объектно-ориентированный и типобезопасный язык программирования общего назначения.

### История и создание

*   **Разработчик:** Компания **Microsoft** под руководством **Андерса Хейлсберга** (создателя Turbo Pascal и Delphi).
*   **Год выпуска:** **2000** (первая публичная preview-версия). Первая стабильная версия вышла в 2002 году вместе с .NET Framework 1.0.
*   **Мотивация:** Ответ Microsoft на растущую популярность платформы **Java**. Целью было создать язык для построения enterprise-приложений, которые могли бы работать в распределенной среде (интернете) и легко взаимодействовать друг с другом.
*   **Платформа:** Изначально и неразрывно связан с платформой **.NET** (сначала .NET Framework, теперь .NET 5/6/7/8+). C# компилируется не в машинный код, а в промежуточный язык (IL, Intermediate Language), который затем выполняется виртуальной машиной (CLR, Common Language Runtime).

### Ключевые особенности и философия

C# унаследовал лучшее от своих предшественников (C++, Java, Delphi) и продолжает активно развиваться, добавляя современные возможности.

1.  **Объектно-ориентированный:** Полная поддержка инкапсуляции, наследования и полиморфизма.
2.  **Типобезопасность:** Компилятор и CLR строго следят за типами данных, предотвращая множество ошибок на этапе компиляции и выполнения.
3.  **Управляемый код:** Память автоматически управляется **Сборщиком мусора (Garbage Collector)**, что избавляет программиста от ручного управления памятью (в отличие от C/C++).
4.  **Компонентно-ориентированный:** Легко создавать и использовать программные компоненты (например, через NuGet — менеджер пакетов для .NET).
5.  **Постоянное развитие:** Команда C# следует философии "**Управляемый код, но без потерь в производительности**". Язык регулярно получает крупные обновления.

### Эволюция C#

*   **C# 1.0 (2002):** Базовый ООП-язык, очень похожий на Java.
*   **C# 2.0 (2005):** **Generics (обобщенные типы)** — одно из важнейших добавлений, повысившее типобезопасность и производительность.
*   **C# 3.0 (2007):** **LINQ (Language Integrated Query)**, лямбда-выражения, анонимные типы. Это изменило способ работы с данными.
*   **C# 5.0 (2012):** **Async/Await** — упрощение асинхронного программирования, критически важно для современных UI- и веб-приложений.
*   **C# 6.0-7.3 (2015-2018):** Улучшения производительности разработчика: интерполяция строк, pattern matching, кортежи, записи (`record`).
*   **C# 8.0+ (2019-н.в.):** Дальнейшее развитие: **nullable reference types** (для борьбы с исключениями `NullReferenceException`), более продвинутый pattern matching, упрощение кода.

### Для чего используется C#?

C# — универсальный язык. Его основная сфера применения сместилась с десктопных приложений (Windows Forms, WPF) в сторону веба и кроссплатформенности.

1.  **Backend-разработка:** **ASP.NET Core** — один из ведущих фреймворков для создания высокопроизводительных веб-API и веб-приложений.
2.  **Игры:** **Unity** — один из самых популярных игровых движков в мире, использующий C# в качестве основного языка скриптинга.
3.  **Десктопные приложения:** Windows-приложения на **Windows Forms**, **WPF**, **UWP** и **MAUI**.
4.  **Мобильные приложения:** С помощью **.NET MAUI** можно разрабатывать кроссплатформенные приложения для iOS и Android.
5.  **Облачные и микросервисные приложения:** Интеграция с **Azure** и поддержка современных архитектурных паттернов.
6.  **Enterprise-приложения:** Крупные корпоративные системы благодаря надежности, безопасности и богатому набору инструментов.

# Правила именования переменных и идентификаторов в C#

## 1. Основные соглашения об именовании (Coding Conventions)

### Стили именования:
- **camelCase**: первое слово с маленькой буквы, каждое последующее с большой (`myVariableName`)
- **PascalCase**: каждое слово с большой буквы (`MyVariableName`)
- **UPPER_CASE**: все буквы заглавные, слова разделяются подчеркиванием (`MY_CONSTANT`)

## 2. Правила для переменных (локальных)

### Обязательные правила:
- ✅ Используйте **camelCase**
- ✅ Имя должно начинаться с буквы или символа `_`
- ✅ После первого символа можно использовать буквы, цифры и `_`
- ✅ Имена чувствительны к регистру (`age` и `Age` - разные переменные)

```csharp
// Правильно
int age = 25;
string firstName = "John";
double _tempValue = 36.6;
int user2Id = 100;

// Неправильно
int Age = 25;        // должно быть camelCase
string firstname = "John"; // не descriptive
double temp-value = 36.6;  // дефис не allowed
int 2userid = 100;         // начинается с цифры
```

### Рекомендации:
- ✔️ Используйте осмысленные имена
- ✔️ Избегайте сокращений (кроме общепринятых)
- ✔️ Для булевых переменных используйте префиксы `is`, `has`, `can`

```csharp
// Хорошо
bool isActive = true;
bool hasPermission = false;
bool canEdit = true;
string customerName = "Alice";

// Плохо
bool a = true;           // не понятное имя
string cn = "Alice";     // непонятное сокращение
bool permission = false; // для bool лучше hasPermission
```

## 3. Правила для идентификаторов (классы, методы, свойства)

### Обязательные правила:
- ✅ Используйте **PascalCase**
- ✅ Имя должно начинаться с буквы
- ✅ Можно использовать буквы, цифры и `_` (но `_` не рекомендуется)

```csharp
// Правильно
public class CustomerOrder {}
public void CalculateTotalPrice() {}
public string FirstName { get; set; }

// Неправильно
public class customer_order {}  // должно быть PascalCase
public void calculate_total() {} // должно быть PascalCase
public string firstname { get; set; } // должно быть PascalCase
```

## 4. Правила для констант

### Обязательные правила:
- ✅ Используйте **UPPER_CASE**
- ✅ Слова разделяются подчеркиванием `_`

```csharp
// Правильно
public const int MAX_USERS = 100;
public const string DEFAULT_FILE_NAME = "config.txt";
public const double PI_VALUE = 3.14159;

// Неправильно
public const int maxUsers = 100;       // должно быть UPPER_CASE
public const string DefaultFileName = "config.txt"; // должно быть UPPER_CASE
```

## 5. Запрещенные имена

### Нельзя использовать ключевые слова C#:
```csharp
// Ошибка компиляции
int class = 10;     // 'class' - ключевое слово
string void = "test"; // 'void' - ключевое слово
double public = 1.0; // 'public' - ключевое слово
```

### Можно использовать с префиксом @:
```csharp
// Разрешено с @
int @class = 10;        // но лучше избегать
string @void = "test";  // используйте alternative имена
```

## 6. Примеры хороших и плохих имен

### Хорошие практики:
```csharp
// Локальные переменные (camelCase)
int itemCount = 5;
string userName = "john_doe";
DateTime birthDate = new DateTime(1990, 5, 15);
bool isValid = true;

// Классы и методы (PascalCase)
public class ShoppingCart {}
public class FileManager {}
public void SaveToDatabase() {}
public string GetFullName() {}

// Константы (UPPER_CASE)
public const int MAX_RETRY_ATTEMPTS = 3;
public const string CONNECTION_STRING = "Server=...";
```

### Плохие практики:
```csharp
// Не descriptive
int a = 10;
string str = "hello";
double d = 3.14;

// Смешение стилей
int Item_count = 5;     // частично Pascal, частично snake_case
string USERname = "john"; // частично UPPER, частично camel

// Слишком длинные
int numberOfElementsInTheArrayThatWeNeedToProcess = 100;

// Сокращения
string custAddr = "123 Main St"; // вместо customerAddress
int empId = 1001;                // вместо employeeId
```


## 7. Важные замечания

1. **Будьте последовательны** - выберите стиль и придерживайтесь его
2. **Имена должны быть понятными** - код должен читаться как книга
3. **Избегайте магических чисел** - используйте константы с понятными именами
4. **Длина имени** - должна отражать scope переменной

```csharp
// Вместо магических чисел
if (status == 1) // что значит 1?

// Лучше использовать константы
const int STATUS_ACTIVE = 1;
const int STATUS_INACTIVE = 2;

if (status == STATUS_ACTIVE)
{
    // логика для активного статуса
}
```
