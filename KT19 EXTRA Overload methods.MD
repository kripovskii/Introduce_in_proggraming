### Перегрузка методов (Method Overloading) в C#

#### Что такое перегрузка методов?

Перегрузка методов — это возможность в одном классе объявить **несколько методов с одинаковым именем**, но с **разными параметрами**. Компилятор C# сам определяет, какой именно метод нужно вызвать, исходя из переданных аргументов. Это называется **полиморфизмом во время компиляции** (compile-time polymorphism).

Перегрузка позволяет сделать интерфейс класса более удобным и интуитивным: пользователь может вызывать один и тот же логический оператор разными способами.

#### Правила перегрузки методов

Чтобы методы считались перегруженными, должны выполняться следующие условия:

1. **Методы должны иметь одинаковое имя**.
2. **Методы должны иметь разный список параметров** по одному из критериев:
   - Разное **количество** параметров.
   - Разные **типы** параметров (даже если количество одинаковое).
   - Разный **порядок** типов параметров.
3. **Возвращаемый тип НЕ участвует** в определении перегрузки.  
   Нельзя перегрузить два метода, отличающиеся только возвращаемым типом.

Примеры правильной перегрузки:

```csharp
public void Print(int value) { ... }
public void Print(string value) { ... }
public void Print(int a, int b) { ... }
public void Print(double value) { ... }
public void Print(string text, int count) { ... }
public void Print(int count, string text) { ... }  // порядок имеет значение!
```

Примеры **неправильной** перегрузки (ошибка компиляции):

```csharp
public int Calculate(int a, int b) { ... }
public double Calculate(int x, int y) { ... }  // только возвращаемый тип отличается → ошибка!

public void Show(string s) { ... }
public void Show(string text) { ... }         // одинаковая сигнатура → ошибка!
```

#### Как компилятор выбирает метод?

Процесс называется **overload resolution** (разрешение перегрузки). Компилятор ищет наиболее подходящий метод по следующим шагам:

1. Точное совпадение типов аргументов.
2. Совпадение после неявных преобразований (например, int → long, int → double).
3. Совпадение через пользовательские преобразования (редко).
4. Если найдено несколько подходящих — ошибка неоднозначности (ambiguous call).

Пример:

```csharp
public void Display(int x) => Console.WriteLine("int: " + x);
public void Display(double x) => Console.WriteLine("double: " + x);
public void Display(object x) => Console.WriteLine("object: " + x);

Display(10);     // вызовет Display(int)
Display(10.5);   // вызовет Display(double)
Display("text"); // вызовет Display(object)
```

#### Практические примеры перегрузки

##### 1. Классический пример — математические операции

```csharp
public static class Calculator
{
    public static int Add(int a, int b) => a + b;

    public static int Add(int a, int b, int c) => a + b + c;

    public static int Add(params int[] numbers)
    {
        int sum = 0;
        foreach (int n in numbers)
            sum += n;
        return sum;
    }
}
```

Вызовы:

```csharp
Console.WriteLine(Calculator.Add(5, 10));          // 15
Console.WriteLine(Calculator.Add(1, 2, 3));        // 6
Console.WriteLine(Calculator.Add(1, 2, 3, 4, 5));  // 15
Console.WriteLine(Calculator.Add());              // 0 (пустой params)
```

##### 2. Перегрузка для разных типов данных

```csharp
public static string Repeat(string text, int count) => 
    string.Concat(Enumerable.Repeat(text, count));

public static string Repeat(char ch, int count) => 
    new string(ch, count);
```

Вызовы:

```csharp
Console.WriteLine(Repeat("Hi", 3));  // HiHiHi
Console.WriteLine(Repeat('*', 10));  // **********
```

##### 3. Перегрузка с параметрами по умолчанию (не путать!)

Важно: **параметры по умолчанию** — это НЕ перегрузка, хотя часто используются вместе.

```csharp
public void Log(string message, bool showTime = true)
{
    if (showTime)
        Console.WriteLine($"[{DateTime.Now}] {message}");
    else
        Console.WriteLine(message);
}
```

Это один метод, но его можно вызывать как `Log("Error")` или `Log("Error", false)`.

#### Преимущества перегрузки методов

- Удобный и читаемый API.
- Единое имя для логически схожих операций.
- Меньше необходимости придумывать разные имена (AddTwoNumbers, AddThreeNumbers и т.д.).
- Лучшая поддержка в IntelliSense.

#### Недостатки и подводные камни

- Может привести к неоднозначности, если перегрузки плохо спроектированы.
- Сложнее отлаживать, если много похожих вариантов.
- При использовании `params` и других перегрузок легко получить неожиданный выбор метода.

Пример неоднозначности:

```csharp
public void Process(object obj) { ... }
public void Process(string s) { ... }

Process(null); // ОШИБКА! Компилятор не знает, какой выбрать (string тоже object)
```

Решение: явное приведение — `Process((string)null);`

#### Перегрузка vs Переопределение (Overriding)

| Характеристика         | Перегрузка (Overloading)             | Переопределение (Overriding)             |
|-------------------------|--------------------------------------|------------------------------------------|
| Где происходит          | В одном классе                       | В иерархии наследования (базовый и производный) |
| Ключевое слово          | Нет                                 | `virtual` / `override`                   |
| Имя метода              | Одинаковое                          | Одинаковое                               |
| Сигнатура параметров    | Разная                              | Одинаковая                               |
| Возвращаемый тип        | Может отличаться                    | Должен совпадать (или ковариантный)      |
| Время связывания        | Компиляция (статический полиморфизм) | Выполнение (динамический полиморфизм)    |

#### Выводы

Перегрузка методов — мощный и часто используемый механизм в C#, который делает код выразительнее и удобнее в использовании. Главное — проектировать перегрузки аккуратно, избегая неоднозначностей и чрезмерного количества вариантов.

В стандартной библиотеке .NET перегрузка используется повсеместно: `Console.WriteLine`, `string.Join`, `Math.Max`, `Enumerable.Where` и многие другие.
