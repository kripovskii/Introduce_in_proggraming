##  Что такое функция (метод) в C#?

**Функция (метод)** — это именованный блок кода, который выполняет определённую задачу. Методы позволяют организовать код, избежать дублирования и упростить поддержку программы.

В C# все методы объявляются **внутри классов** — язык полностью объектно-ориентированный.

---

##  Синтаксис объявления метода

```csharp
[модификатор_доступа] [модификаторы] возвращаемый_тип ИмяМетода([параметры])
{
    // тело метода
}
```

### Примеры компонентов:

- **Модификаторы доступа**: `public`, `private`, `protected`, `internal`
- **Возвращаемый тип**: `void`, `int`, `string`, `bool`, любой тип
- **Имя метода**: по соглашению — `PascalCase`
- **Параметры**: `(int x, string name)` — опционально

---

##  Примеры методов

### 1. Метод без параметров и без возврата (`void`)
```csharp
public void SayHello()
{
    Console.WriteLine("Привет, мир!");
}
```

### 2. Метод с параметрами и возвратом значения
```csharp
public int Add(int a, int b)
{
    return a + b;
}
```

### 3. Метод с параметром по умолчанию
```csharp
public void Greet(string name = "Гость")
{
    Console.WriteLine($"Привет, {name}!");
}
```

Вызов:
```csharp
Greet();          // Выведет: Привет, Гость!
Greet("Анна");    // Выведет: Привет, Анна!
```

### 4. Метод с параметром `out`
```csharp
public bool TryParseInt(string input, out int result)
{
    return int.TryParse(input, out result);
}
```

Использование:
```csharp
if (TryParseInt("123", out int number))
    Console.WriteLine(number); // 123
```

##  Виды методов

| Тип метода          | Описание |
|---------------------|---------|
| **Статический (static)** | Принадлежит классу, а не экземпляру. Вызывается без создания объекта. |
| **Экземплярный**    | Принадлежит конкретному объекту. |
| **Виртуальный (virtual)** | Может быть переопределён в дочернем классе (`override`). |
| **Абстрактный (abstract)** | Не имеет реализации; обязан быть переопределён в наследнике. |
| **Локальные функции** | Определяются внутри другого метода (начиная с C# 7.0). |

### Пример статического метода:
```csharp
public static double CircleArea(double radius)
{
    return Math.PI * radius * radius;
}
```

Вызов:
```csharp
double area = CircleArea(5.0);
```

##  Перегрузка методов (Method Overloading)

Можно создавать несколько методов **с одинаковым именем**, но **разной сигнатурой** (разное число или типы параметров).

```csharp
public void Print(int value) => Console.WriteLine($"Число: {value}");
public void Print(string text) => Console.WriteLine($"Текст: {text}");
```

Это — **полиморфизм времени компиляции**.

---

## Рекурсия

Метод может вызывать сам себя:

```csharp
public int Factorial(int n)
{
    if (n <= 1) return 1;
    return n * Factorial(n - 1);
}
```

⚠️ Важно: всегда должен быть **базовый случай**, чтобы избежать бесконечной рекурсии.

---

## Практические советы

- Используйте `void`, только если метод **ничего не возвращает**.
- Имена методов — глаголы: `CalculateTotal`, `SaveToFile`, `ValidateInput`.
- Старайтесь, чтобы метод делал **одну задачу** (принцип единственной ответственности).
- Избегайте слишком длинных методов — разбивайте на более мелкие.

## Заключение

Функции (методы) — основа структурированного и читаемого кода в C#. Они позволяют:
- Повторно использовать код
- Упрощать логику программы
- Обеспечивать чёткое разделение ответственности
