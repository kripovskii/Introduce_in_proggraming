# Создание консольного приложения .NET Framework 4.7 в Visual Studio

##  Введение

Создание консольного приложения на .NET Framework 4.7 в Visual Studio — это основа для многих типов программ, от утилит до бэкенд-сервисов. Этот гайд шаг за шагом проведет вас через весь процесс — от установки необходимых компонентов до запуска готового приложения.

## Предварительные требования

Перед началом работы убедитесь, что у вас установлены:

- **Visual Studio 2022** или более ранняя версия (поддерживающая .NET Framework 4.7)
- **.NET Framework 4.7 Developer Pack** (если не входит в состав VS)
- **Рабочая нагрузка "Разработка классических приложений .NET"** в Visual Studio

*Проверить и установить рабочие нагрузки можно через меню Visual Studio: **Инструменты > Получить средства и функции**.*

---

##  Шаг 1: Создание нового проекта

1.  **Запустите Visual Studio**.
2.  На начальном экране выберите **"Создать проект"** (Create a new project).
3.  В открывшемся окне **"Создание нового проекта"**:
    - В поле поиска введите `Console` или `Консоль`.
    - В фильтрах выберите:
        - **Язык:** C#
        - **Платформа:** Windows
        - **Тип проекта:** Консоль
    - Из списка шаблонов выберите **"Консольное приложение (.NET Framework)"** (а не .NET Core или .NET 5/6/7/8). *Важно выбрать именно шаблон с указанием .NET Framework.*

!["Выбор шаблона консольного приложения .NET Framework"](https://placehold.co/600x400/white/grey?text=Console+App+.NET+Framework+Template)

4.  Нажмите кнопку **"Далее"**.

---

##  Шаг 2: Настройка проекта

1.  В окне **"Настройка нового проекта"**:
    - **Имя проекта:** Введите понятное имя, например, `MyConsoleApp`.
    - **Расположение:** Укажите путь, где будет сохранен ваш проект.
    - **Имя решения:** Обычно совпадает с именем проекта.
2.  **Важный момент:** Убедитесь, что в поле **"Платформа"** или **"Версия .NET"** выбрана именно **.NET Framework 4.7** (или более новая, например, 4.7.1, 4.7.2, 4.8). Если нужной версии нет в списке, убедитесь, что она установлена в системе.

!["Настройка проекта и выбор .NET Framework 4.7"](https://placehold.co/600x400/white/grey?text=Configure+Project+.NET+Framework+4.7)

3.  Нажмите кнопку **"Создать"**.

---

##  Шаг 3: Обзор структуры проекта

После создания проекта Visual Studio автоматически сгенерирует шаблонный код. В **Обозревателе решений** (Solution Explorer) вы увидите следующую структуру:

- **Ссылки (References):** Сборки .NET Framework, от которых зависит ваш проект.
- **App.config:** Файл конфигурации приложения (пока можно не менять).
- **Program.cs:** Главный файл с исходным кодом. Именно здесь вы будете писать свой код.

!["Структура проекта в Обозревателе решений"](https://placehold.co/600x400/white/grey?text=Solution+Explorer+Structure)

*Откройте файл `Program.cs`, дважды щелкнув по нему в Обозревателе решений.*

---

##  Шаг 4: Знакомство с кодом

Visual Studio создаст файл `Program.cs` со следующим стандартным кодом:

```csharp
using System;
using System.Collections.Generic;
using System.Linq;
using System.Text;
using System.Threading.Tasks;

namespace MyConsoleApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello World!");
        }
    }
}
```


  **Запустите приложение**:
    - Нажмите **Ctrl + F5** (запуск без отладки). *Это предпочтительный способ, так как консольное окно не закроется сразу после выполнения.*
    - Или нажмите **F5** (запуск с отладкой). В этом случае консоль закроется сразу после завершения работы кода, если не добавить `Console.ReadKey()`.

## ❌ Распространенные ошибки и устранение проблем

| Проблема | Решение |
| :--- | :--- |
| **Не удается найти шаблон .NET Framework** | Установите рабочую нагрузку **".NET desktop development"** через Visual Studio Installer. |
| **В списке целевых версий нет .NET 4.7** | Убедитесь, что **.NET Framework 4.7 Developer Pack** установлен в системе. |
| **Консольное окно сразу закрывается** | Запускайте приложение через **Ctrl + F5** или добавьте `Console.ReadKey()` в конец метода `Main`. |
| **Ошибка "Не удается найти сборку"** | Проверьте, правильно ли выбрана целевая версия .NET Framework в свойствах проекта и установлена ли она на целевом компьютере. |
