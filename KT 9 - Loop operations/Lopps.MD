# Лекция: Циклы в C# — подробное описание с примерами

**Кратко:** циклы — основной инструмент для многократного выполнения кода. В C# есть четыре основных формы циклов: `for`, `while`, `do..while` и `foreach`. Также важны управляющие операторы `break` / `continue` и работа с вложенными циклами. Ниже — объяснение, практические примеры, частые ошибки и задания для практики.

---

# 1. Когда и зачем использовать циклы
Циклы применяются, когда нужно повторять однотипную операцию много раз:
- итерировать элементы коллекции (массив, `List<T>`, `Dictionary<TKey,TValue>`);
- выполнять вычисления с накоплением (сумма, максимум, подсчёт);
- пока не выполнится некоторое условие (чтение ввода, ожидание события);
- генерировать выходные данные (таблицы, матрицы).

---

# 2. `for` — классический индексный цикл

**Синтаксис**
```csharp
for (инициализация; условие; итерация)
{
    // тело
}
```

**Пример — суммирование элементов массива**
```csharp
int[] a = { 2, 5, 7, 3 };
int sum = 0;
for (int i = 0; i < a.Length; i++)
{
    sum += a[i];
}
Console.WriteLine($"Sum = {sum}"); // Sum = 17
```

**Особенности и советы**
- Полезен, когда нужен индекс (i) или модификация коллекции по индексу.
- Внимание на границы: `i < a.Length` — обычный корректный шаблон. `i <= a.Length - 1` эквивалентен, но легче ошибиться.
- Можно инициализировать/изменять несколько переменных: `for (int i = 0, j = n - 1; i < j; i++, j--) { ... }`.

**Пример — обход в обратном порядке**
```csharp
for (int i = a.Length - 1; i >= 0; i--)
    Console.WriteLine(a[i]);
```

# 3. `while` — цикл с условием в начале

**Синтаксис**
```csharp
while (условие)
{
    // тело
}
```

**Пример — чтение чисел до нуля**
```csharp
int x;
int sum = 0;
Console.WriteLine("Введите числа (0 — конец):");
while (int.TryParse(Console.ReadLine(), out x) && x != 0)
{
    sum += x;
}
Console.WriteLine($"Sum = {sum}");
```

**Особенности**
- Условие проверяется перед выполнением тела.
- Риск бесконечного цикла, если условие никогда не станет `false`.

---

# 4. `do..while` — условие в конце (выполняется минимум один раз)

**Синтаксис**
```csharp
do
{
    // тело
} while (условие);
```

**Пример**
```csharp
int n;
do
{
    Console.Write("Введите положительное число: ");
    n = int.Parse(Console.ReadLine());
} while (n <= 0);
```

# 5. `foreach` — итерация по коллекции (удобно и безопасно)

**Синтаксис**
```csharp
foreach (var item in collection)
{
    // использовать item
}
```

**Примеры**
```csharp
var list = new List<string> { "apple", "banana", "cherry" };
foreach (var s in list)
    Console.WriteLine(s);

// Работа со словарём
var dict = new Dictionary<string, int> { ["a"] = 1, ["b"] = 2 };
foreach (var kvp in dict)
    Console.WriteLine($"{kvp.Key} -> {kvp.Value}");
```

**Важно:** нельзя изменять коллекцию (добавлять/удалять элементы) во время `foreach` — это вызовет `InvalidOperationException`.  
Безопасные альтернативы удаления:
- использовать `RemoveAll` (для `List<T>`),
- проходить в обратном порядке с `for` и `RemoveAt`,
- собирать элементы для удаления в отдельный список, затем удалить их.

**Пример: безопасное удаление чётных из List**
```csharp
var nums = new List<int> {1,2,3,4,5,6};
// вариант 1: RemoveAll
nums.RemoveAll(x => x % 2 == 0);

// вариант 2: проход в обратном порядке
for (int i = nums.Count - 1; i >= 0; i--)
    if (nums[i] % 2 == 0)
        nums.RemoveAt(i);
```

# 6. `break` и `continue`

- `break` — немедленно прерывает текущий цикл (переходит к следующему оператору после цикла).
- `continue` — прекращает текущую итерацию и переходит к следующей (в `for` выполнит шаг итерации).

**Примеры**
```csharp
// break
for (int i = 0; i < 10; i++)
{
    if (i == 5) break; // цикл завершится, когда i==5
    Console.WriteLine(i);
}

// continue
for (int i = 0; i < 10; i++)
{
    if (i % 2 == 0) continue; // пропускаем чётные
    Console.WriteLine(i); // печатает только нечетные
}
```

# 7. Вложенные циклы и выход из них

**Пример — таблица умножения**
```csharp
for (int i = 1; i <= 5; i++)
{
    for (int j = 1; j <= 5; j++)
    {
        Console.Write($"{i * j,3}");
    }
    Console.WriteLine();
}
```

**Выход из нескольких уровней**
- `break` выходит только из ближайшего цикла.
- Для выхода из внешнего цикла можно:
  - использовать флаг:
    ```csharp
    bool found = false;
    for (...)
    {
        for (...)
        {
            if (condition) { found = true; break; }
        }
        if (found) break;
    }
    ```
  - использовать `goto метка:` (редко рекомендуется):
    ```csharp
    for (...) {
      for (...) {
        if (condition) goto Found;
      }
    }
    Found:
    Console.WriteLine("found");
    ```
  - вернуть управление из метода (`return`) — если уместно.

# 8. Практические примеры (полезные задачи)

**1) Нахождение максимума**
```csharp
int[] arr = { 3, 7, 2, 9, 4 };
int max = arr[0];
for (int i = 1; i < arr.Length; i++)
    if (arr[i] > max) max = arr[i];
Console.WriteLine($"Max = {max}"); // Max = 9
```

**2) Проверка на простое число (простая реализация)**
```csharp
bool IsPrime(int n)
{
    if (n <= 1) return false;
    if (n <= 3) return true;
    for (int i = 2; i * i <= n; i++)
        if (n % i == 0) return false;
    return true;
}
Console.WriteLine(IsPrime(37)); // True
```

**3) Факториал с циклом**
```csharp
long Factorial(int n)
{
    long res = 1;
    for (int i = 2; i <= n; i++)
        res *= i;
    return res;
}
Console.WriteLine(Factorial(10)); // 3628800
```

**4) Перебор с индексом в `foreach` (через LINQ)**
```csharp
using System.Linq;
var list = new[] {"a","b","c"};
foreach (var pair in list.Select((item, index) => new {item, index}))
{
    Console.WriteLine($"{pair.index}: {pair.item}");
}
```

# 9. Частые ошибки и «подводные камни»

1. **Бесконечные циклы** — забудете изменять условие (например, `while(true)` без `break`).
2. **Off-by-one** — неправильное условие (`<=` вместо `<`).
3. **Модификация коллекции во время `foreach`** — вызывает `InvalidOperationException`.
4. **Плохая производительность** — вложенные циклы с большими размерами (`O(n*m)`) могут быть дорогими.
5. **Побочные эффекты в параллельных циклах** — при использовании `Parallel.For` будьте осторожны с разделяемыми переменными.
6. **Индексы и типы** — при большом количестве элементов индекс `int` обычно OK (до ~2 млрд), но для бесконечно больших данных нужны другие подходы/потоки.

# 10. Советы по стилю и когда что выбирать

- Используйте `foreach`, когда нужно просто пройти все элементы и **не** нужен индекс и не предполагается изменение коллекции.
- Используйте `for`, если нужен индекс, доступ по индексу или нужна модификация структуры через индекс.
- `while`/`do..while` — когда количество итераций заранее неизвестен и зависит от внешнего условия (ввод, состояние).
- Для фильтрации/агрегации изучите LINQ (`Where`, `Select`, `Sum`), он часто делает код короче и выразительнее.
- Для CPU-bound задач, где итерации независимы, можно рассмотреть `Parallel.For` или PLINQ, но только после профилирования.

# 11. Небольшие упражнения (с ответами)

**Упражнение 1.** Выведите числа Фибоначчи до 1000.  
**Решение (идея):** используйте `while` или `for`, генерируя `a`, `b`, пока `b <= 1000`.

**Пример решения**
```csharp
int a = 0, b = 1;
while (b <= 1000)
{
    Console.WriteLine(b);
    int tmp = a + b;
    a = b;
    b = tmp;
}
```

**Упражнение 2.** Дан `List<int>`, удалить все отрицательные числа.  
**Решение:** `list.RemoveAll(x => x < 0);` или проход в обратном порядке и `RemoveAt`.

**Упражнение 3.** Найти индекс первого элемента, который больше заданного числа `x`.  
**Решение:**
```csharp
int IndexOfFirstGreater(int[] a, int x)
{
    for (int i = 0; i < a.Length; i++)
        if (a[i] > x) return i;
    return -1; // не найден
}
```

# 12. Подведение итогов
- Циклы — базовый инструмент, нужно знать синтаксис `for`, `while`, `do..while`, `foreach`.
- Выбирайте тип цикла исходя из задачи: нужен ли индекс, нужно ли модифицировать коллекцию, заранее известен ли размер.
- Будьте внимательны с границами и модификацией коллекций во время итерации.
- Практикуйтесь: реализуйте классические задачи (факториал, простое число, поиск, фильтрация) в нескольких вариантах (`for`, `foreach`, LINQ`) — это укрепит понимание.
