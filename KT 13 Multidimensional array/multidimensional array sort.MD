# Лекция: Сортировка многомерных массивов в C#

## Введение

Сортировка — одна из фундаментальных задач в информатике, которая заключается в упорядочивании элементов по определённому критерию. В C# сортировка одномерных массивов реализована в стандартной библиотеке (например, метод `Array.Sort()`), но сортировка **многомерных массивов** требует особого подхода, так как стандартные методы не поддерживают прямую сортировку многомерных структур.  

В этой лекции мы рассмотрим:
- Особенности многомерных массивов в C#;
- Почему стандартные методы сортировки не подходят;
- Алгоритмические подходы к сортировке многомерных массивов;
- Практические примеры реализации.

---

## 1. Многомерные массивы в C#

В C# существуют два типа многомерных массивов:
- **Прямоугольные массивы** (`int[,]`) — все строки имеют одинаковую длину.
- **"Зубчатые" (jagged) массивы** (`int[][]`) — массив массивов, где подмассивы могут иметь разную длину.

Примеры:
```csharp
// Прямоугольный массив 3x4
int[,] rectArray = new int[3, 4];

// Зубчатый массив
int[][] jaggedArray = new int[3][];
jaggedArray[0] = new int[2];
jaggedArray[1] = new int[5];
jaggedArray[2] = new int[3];
```

**Важно:**  
Метод `Array.Sort()` работает **только с одномерными массивами**. Поэтому, чтобы отсортировать многомерный массив, его нужно либо преобразовать, либо реализовать собственную логику сортировки.

---

## 2. Основные стратегии сортировки многомерных массивов

Существует несколько подходов:

### Стратегия 1: Сортировка строк (или столбцов) как отдельных одномерных массивов

Если задача — отсортировать каждую строку или каждый столбец независимо, то можно применить `Array.Sort()` к каждой строке отдельно.

**Пример (прямоугольный массив, сортировка строк):**
```csharp
int[,] matrix = {
    {5, 2, 8},
    {1, 9, 3},
    {7, 4, 6}
};

int rows = matrix.GetLength(0);
int cols = matrix.GetLength(1);

for (int i = 0; i < rows; i++)
{
    // Извлекаем строку
    int[] row = new int[cols];
    for (int j = 0; j < cols; j++)
        row[j] = matrix[i, j];

    // Сортируем
    Array.Sort(row);

    // Записываем обратно
    for (int j = 0; j < cols; j++)
        matrix[i, j] = row[j];
}
```

> ⚠️ Этот подход **не сохраняет связи между столбцами**. Если строки представляют записи (например, `[Имя, Возраст, Зарплата]`), то сортировка по отдельным строкам нарушит целостность данных.

---

### Стратегия 2: Сортировка строк как записей по одному ключу
> Более подробно этот способ разберем на 2м курсе

Часто многомерный массив используется для хранения **таблицы данных**, где каждая строка — это запись. В этом случае нужно сортировать **всю строку целиком**, используя значение определённого столбца как ключ.

Для этого:
1. Преобразуем строки в объекты (например, кортежи или `ValueTuple`);
2. Сортируем список объектов;
3. Обратно преобразуем в массив.

**Пример (сортировка по второму столбцу):**
```csharp
int[,] data = {
    {10, 30, 50},
    {20, 10, 40},
    {30, 20, 60}
};

int rows = data.GetLength(0);
int cols = data.GetLength(1);

// Шаг 1: Преобразуем в список кортежей
var list = new List<(int[] row, int key)>();
for (int i = 0; i < rows; i++)
{
    int[] row = new int[cols];
    for (int j = 0; j < cols; j++)
        row[j] = data[i, j];
    list.Add((row, row[1])); // ключ — второй столбец (индекс 1)
}

// Шаг 2: Сортируем по ключу
list.Sort((a, b) => a.key.CompareTo(b.key));

// Шаг 3: Записываем обратно
for (int i = 0; i < rows; i++)
{
    for (int j = 0; j < cols; j++)
        data[i, j] = list[i].row[j];
}
```

### Стратегия 3: Реализация собственного алгоритма сортировки "на месте"

Можно реализовать, например, **пузырьковую сортировку строк** прямо в многомерном массиве, сравнивая ключевые элементы.

**Пример (пузырьковая сортировка по первому столбцу):**
```csharp
int rows = data.GetLength(0);
int cols = data.GetLength(1);
int keyColumn = 0; // сортируем по первому столбцу

for (int i = 0; i < rows - 1; i++)
{
    for (int j = 0; j < rows - i - 1; j++)
    {
        if (data[j, keyColumn] > data[j + 1, keyColumn])
        {
            // Обмен строками
            for (int k = 0; k < cols; k++)
            {
                int temp = data[j, k];
                data[j, k] = data[j + 1, k];
                data[j + 1, k] = temp;
            }
        }
    }
}
```

> ✅ Преимущество: работает **на месте**, без дополнительной памяти.  
> ❌ Недостаток: сложность O(n²), неэффективно для больших данных.

## 3. Сортировка зубчатых массивов (`int[][]`)

Для зубчатых массивов сортировка проще, так как каждая строка — это отдельный объект (`int[]`). Можно использовать `LINQ`
(позволяет писать SQL-подобные запросы непосредственно в коде C# для работы с данными из различных источников: массивов, списков, баз данных, XML-документов и других коллекций):


```csharp
int[][] jagged = {
    new int[] {5, 2, 8},
    new int[] {1, 9, 3},
    new int[] {7, 4, 6}
};

// Сортировка строк как записей по первому элементу
Array.Sort(jagged, (a, b) => a[0].CompareTo(b[0]));
```

> ⚠️ Убедитесь, что все подмассивы содержат нужный индекс (например, `[0]`).

---

## 4. Рекомендации и лучшие практики

1. **Избегайте прямоугольных массивов для табличных данных.** Лучше использовать:
   - `List<T>` с пользовательским классом/рекордом;
   - `List<ValueTuple<...>>`;
   - `DataTable` (в desktop-приложениях).

2. **Используйте LINQ для простых случаев:**
   ```csharp
   var sorted = original.Select(row => row.ToArray())
                       .OrderBy(row => row[1])
                       .ToArray();
   ```

3. **Для больших данных — применяйте эффективные алгоритмы** (QuickSort, MergeSort), а не пузырьковую сортировку.

4. **Учитывайте память:** преобразование в одномерный массив и обратно может быть затратным.

---

## Заключение

Сортировка многомерных массивов в C# — это не встроенная операция, а **алгоритмическая задача**, требующая понимания структуры данных и целей сортировки.

Ключевые моменты:
- Определите, что именно нужно отсортировать: строки, столбцы или весь массив как плоскую структуру;
- Выберите подходящий метод: преобразование + `Array.Sort()`, собственный алгоритм, или переход на более гибкие структуры данных;
- Учитывайте производительность и целостность данных.

>  **Совет:** В большинстве реальных приложений вместо `int[,]` лучше использовать коллекции объектов — это упрощает сортировку, фильтрацию и поддержку кода.
